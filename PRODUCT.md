# Product

## Register

product

## Users

Team administrators and members who use a shared Claude Code WebUI to run, review, and manage agent sessions against controlled workspaces.

## Product Purpose

Provide a deployable team console for Claude Code sessions, workspace configuration, MCP tool approvals, user management, and audit visibility. Success means users can find the right session quickly, understand agent state without noise, and continue work without disturbing the host Claude Code configuration.

## Brand Personality

Calm, operational, precise. The interface should feel like a reliable internal workbench rather than a marketing surface.

## Anti-references

Avoid decorative dashboards, oversized hero sections inside the authenticated app, modal-heavy workflows, dense walls of unstructured logs, and visual styling that hides standard controls.

## Design Principles

- Keep repeated work fast: navigation, session search, approvals, and message review should stay close to the primary workspace.
- Make status scannable: running, completed, waiting, failed, and ownership states should be obvious at a glance.
- Preserve user context: live updates must not erase drafts, reset scroll, or collapse the user's local view unexpectedly.
- Prefer inline management over blocking modals when the task is review or approval.
- Stay restrained: use color for state, selection, and primary actions.

## Accessibility & Inclusion

Target accessible contrast, keyboard-reachable controls, visible focus states, reduced-motion compatibility, and layouts that remain usable on narrower laptop screens.
